## IT Documentation — Runbook / SOP / KB Article

**Ask me:**
1. Document type? (SOP, runbook, KB article, incident report, change log, onboarding guide)
2. What process or topic does it cover?
3. Who's the audience? (IT staff, end users, management, vendor)
4. Any specific steps, tools, or warnings to include?

---

**Format standards:**
- Title + version + last updated date
- Overview (1–2 sentences: what this doc covers and when to use it)
- Prerequisites (what you need before starting)
- Step-by-step procedure (numbered, portal steps first)
- ⚠️ Warnings for any destructive steps
- Verification steps (how to confirm it worked)
- Escalation path (who to contact if stuck)
